\section{Lose Conditions}

\subsection{Move Limit System}

The player loses if they exceed the move limit before winning.

\textbf{Base limit:} 28 moves \\
\textbf{With coffee:} 33 moves \\
\textbf{Optimal solution:} 20 moves \\
\textbf{Buffer:} 8-13 moves for exploration

\subsection{CRITICAL: What Counts as a Move?}

This is the most important mechanic in the game. ONLY movement commands count toward the limit.

\textbf{Commands that COUNT (use 1 move):}
\begin{itemize}
\item \texttt{go north/south/east/west/up/down}
\end{itemize}

\textbf{Commands that are FREE (0 moves):}
\begin{itemize}
\item \texttt{look} - View description
\item \texttt{inventory} - Check items
\item \texttt{score} - See progress
\item \texttt{log} - View history
\item \texttt{take [item]} - Pick up
\item \texttt{drop [item]} - Drop
\item \texttt{examine [item]} - Inspect
\item \texttt{talk to friend} - Dialogue
\item \texttt{enter code} - Puzzle
\item \texttt{drink coffee} - Consume
\item \texttt{quit, help} - Meta
\end{itemize}

\subsection{Code Implementation}

\textbf{File:} \texttt{adventure.py}

\textbf{Constant:} \texttt{BASE\_MAX\_MOVES = 28} (line 18)

\textbf{Variable:} \texttt{self.max\_moves} (increases with coffee)

\textbf{Tracker:} \texttt{Player.moves\_made} in \texttt{game\_entities.py}

\textbf{Increment:} ONLY \texttt{handle\_go\_command()} calls \texttt{player.increment\_moves()}

\textbf{Check:} \texttt{check\_lose\_condition()} returns \texttt{player.moves\_made >= self.max\_moves}

\section{Enhancements}

\subsection{Enhancement 1: Dynamic Stage Messages (HIGH Complexity)}

\textbf{Complexity: HIGH}

The game displays different contextual messages based on your progress stage.

\textbf{Game Stages:}
\begin{enumerate}
\item \textbf{Start}: Just beginning, haven't collected any items
\item \textbf{Exploring}: Collected first sacred item
\item \textbf{Gathering}: Working on collecting all 3 items
\item \textbf{Ready Ritual}: All 3 items collected, ritual instructions appear
\item \textbf{Ritual Done}: Ritual complete, backup USB acquired
\item \textbf{Winning}: Taking backup USB home
\end{enumerate}

\textbf{Stage-Specific Messages:}
\begin{itemize}
\item \textbf{King's College Circle}: Different message for each stage
  \begin{itemize}
  \item Start: "The circle feels alive... waiting"
  \item Gathering: "You have X of 3 items. Circle pulses..."
  \item Ready: Full ritual instructions display!
  \end{itemize}
\item \textbf{Item Collection}: Progress tracking, remaining items listed
\item \textbf{Gerstein Library}: Partner's sign changes based on stage
\item \textbf{Server Room}: Full story revelation when accessed
\end{itemize}

\textbf{Why HIGH Complexity:}
\begin{itemize}
\item Multi-stage state tracking
\item Context-aware message system
\item Dynamic instruction display
\item Progress monitoring (items collected)
\item Integration with game flow
\item Enhances player guidance
\item Creates narrative progression
\end{itemize}

\textbf{Code:} \texttt{check\_and\_display\_stage\_messages()}, \texttt{game\_stage}, \texttt{items\_collected}

\subsection{Enhancement 2: Summoning Ritual (HIGH Complexity)}

\textbf{Complexity: HIGH}

Horror-movie-inspired ritual where items DISSOLVE and merge into backup USB.

\textbf{The Ritual:}
\begin{itemize}
\item Drop all 3 items at King's College Circle
\item Dramatic summoning ceremony (ASCII art)
\item Items dissolve into particles of light
\item Merge into the backup USB
\item Backup contains EVERYTHING (code + items' power)
\item Just take backup USB home to win!
\end{itemize}

\textbf{Why HIGH:}
\begin{itemize}
\item Multi-item validation
\item Dynamic item spawning
\item Item removal (dissolution effect)
\item Simplified win condition
\item Narrative integration
\end{itemize}

\textbf{Code:} \texttt{check\_ritual()}, \texttt{ritual\_complete}
